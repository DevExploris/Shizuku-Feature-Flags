Introduction

Shizuku is a lightweight Symfony bundle that stores feature flags in your database via Doctrine ORM. No external service, no YAML file to deploy — just a console command away.

Zero config
Works out of the box. Cache and Profiler wire automatically when the relevant Symfony packages are present.
Full lifecycle
Create · Enable · Disable · Lock · Delete with explicit guard rails at each step.
Dev tooling
Symfony Profiler panel, Twig feature() function, cache TTL, and caller-chain tracking.

Usage

PHP

Inject FeatureFlagService and call isEnabled(). An unknown flag always returns false — no exception.

use Devexploris\ShizukuFeatureFlags\Service\FeatureFlagService;

class MyService
{
    public function __construct(private FeatureFlagService $flags) {}

    public function doSomething(): void
    {
        if ($this->flags->isEnabled('my_feature')) {
            // new behaviour
        }
    }
}

Twig

The bundle registers a global feature() function. Unknown flags silently return false.

{% if feature('my_feature') %}
    {# new behaviour #}
{% endif %}

{# inline — e.g. toggling a CSS class #}
<body class="{{ feature('dark_mode') ? 'dark' : 'light' }}">
Unknown flag? If a flag is absent from the database, both isEnabled() and feature() return false. The Symfony Profiler highlights unknown flags so you never silently miss a typo in a flag name.
05 —

Commands

Flag names must be snake_case: lowercase letters, digits, and underscores — starting with a letter.

List flags

$ php bin/console shizuku:list [filter]

filter is optional — All, Enabled, Disabled, Locked. When omitted, an interactive prompt is shown.

Create

$ php bin/console shizuku:flag:create
$ php bin/console shizuku:flag:create --name=my_feature --description="My feature" --enable

Flags are created disabled by default. Pass --enable to start them enabled.

Enable / Disable

$ php bin/console shizuku:flag:enable --name=my_feature
$ php bin/console shizuku:flag:disable --name=my_feature

Locked flags cannot be enabled or disabled.

Lock

$ php bin/console shizuku:flag:lock --name=my_feature

Locking is a one-way operation. It signals that the flag's state is final and the code paths it guards should be removed before deletion.

Delete

$ php bin/console shizuku:flag:delete --name=my_feature
$ php bin/console shizuku:flag:delete --name=my_feature --force

Only locked flags can be deleted by default. Use --force to bypass (asks for confirmation).

01A flag is created disabled by default (or enabled with --enable).

07 —

Caching

When cache.app is available in the container, the bundle wires it automatically. Flag state is cached for 300 seconds and invalidated immediately on every mutating command (enable, disable, lock, delete, create).

Custom cache pool

# config/services.yaml
Devexploris\ShizukuFeatureFlags\Service\FeatureFlagService:
    arguments:
        $cache: '@cache.my_custom_pool'

Disable caching

# config/services.yaml
Devexploris\ShizukuFeatureFlags\Service\FeatureFlagService:
    arguments:
        $cache: ~
Direct database edits bypass cache invalidation. Changes will become visible after the TTL (300 s) expires.

Symfony Profiler

When symfony/web-profiler-bundle is installed, a Feature Flags panel is added to the toolbar automatically — no configuration needed.

Panel itemDescription
Checked / enabled Count of all isEnabled() calls and how many returned true.
Locked flags Flags frozen pending code cleanup — shown as "Must be cleaned before removal".
Unknown flags Flag names checked in code but absent from the database. Toolbar highlights in warning colour.
Caller chain Full Class::method chain that triggered each isEnabled() call, vendor frames excluded.
